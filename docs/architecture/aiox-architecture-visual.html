Como tudo se conecta

Uma visão geral de como o AIOX funciona — para entender antes de mergulhar nos detalhes.

🎯 O que é o AIOX?

AIOX é um sistema operacional de desenvolvimento com IA. Em vez de usar Claude/GPT como um chatbot genérico, o AIOX organiza agentes especializados (Dev, QA, Security, DevOps...) que trabalham juntos em histórias, aprendem com os erros e carregam esse conhecimento para o próximo projeto.

📦 3 Repositórios

AutoPost tem 3 repos no GitHub: o AIOX (framework), o Backend (Python/FastAPI) e o Frontend (Next.js). Cada um com seu propósito.

🤖 13 Agentes Especializados

Cada agente tem um papel: @dev implementa, @qa revisa, @security audita, @devops faz deploy. Cada um com memória e comandos próprios.

🧠 Memória Persistente

O sistema aprende e nunca esquece. Erros, decisões e padrões ficam no Segundo Cérebro e alimentam o próximo projeto automaticamente.

🔄 Reutilização (IDS)

Antes de criar qualquer coisa nova, o sistema verifica se já existe. REUSE → ADAPT → CREATE. Evita reinventar a roda entre projetos.

🔄 Como os repos se comunicam

O repo AIOX não controla backend ou frontend via git. Ele funciona como a "inteligência de desenvolvimento":

  • Stories no AIOX → @dev implementa código no backend/frontend
  • GitHub Actions nos repos de app → CI/CD independente
  • @devops faz push para cada repo separadamente
  • Relatórios de QA salvos no AIOX para rastreabilidade

Dentro do Repositório AIOX

O repo AIOX tem uma estrutura em camadas — cada camada com um nível diferente de proteção e mutabilidade.

🏗️ As 4 Camadas (L1 → L4)

Imagine como um software: tem o núcleo que nunca muda, templates que raramente mudam, configurações do projeto, e o trabalho do dia-a-dia.

L1 — Framework Core 🔒 NUNCA MODIFICAR
.aios-core/core/ · .aios-core/constitution.md · bin/aios.js
O "kernel" do AIOX. A Constituição com os princípios inegociáveis. Modificar aqui quebra tudo.
⚙️
L2 — Framework Templates ⚠️ EXTEND-ONLY
.aios-core/development/tasks/ · templates/ · workflows/
Tasks e templates reutilizáveis. Podem ser adaptados (<30% mudança) por stories de framework como a 7.2 que acabamos de implementar.
🧩
L3 — Project Config ✏️ MODIFICÁVEL (c/ cuidado)
.aios-core/data/ · agents/*/MEMORY.md · core-config.yaml
Configuração específica do projeto. MEMORY.md de cada agente fica aqui — é onde o conhecimento acumulado vive.
⚙️
L4 — Project Runtime 🟢 SEMPRE MODIFICAR
docs/stories/ · docs/qa/ · docs/architecture/
O trabalho do dia-a-dia. Stories, relatórios QA, documentação do projeto. Tudo que o @dev cria e o @qa revisa.

O Segundo Cérebro

O sistema de memória persistente que garante que nenhum aprendizado se perde entre sessões ou projetos.

🧠 Por que existe?

LLMs (Claude, GPT) têm janela de contexto limitada e esquecem tudo ao final de cada sessão. O Segundo Cérebro resolve isso: é uma memória externa, estruturada, que é lida no início de cada sessão e alimentada durante o trabalho.

O Aprendizado Cíclico

Como o conhecimento flui desde uma implementação até virar padrão global — e alimentar o próximo projeto.

🔄 O Grande Ciclo

Cada projeto implementado contribui conhecimento para o sistema. Esse conhecimento é promovido automaticamente para o nível global uma vez por mês — tornando todos os projetos futuros mais inteligentes desde o primeiro dia.

1
💻 @dev implementa uma feature
Encontra um bug difícil, toma uma decisão de arquitetura, descobre um padrão que funciona bem no stack específico do projeto.
Exemplo: "Celery+asyncpg precisa NullPool — conexões persistentes causam deadlock"
2
📝 Registra no Segundo Cérebro Local
Via *cerebro-update "descoberta" — o agente adiciona a entrada em APRENDIZADOS.md com contexto, descoberta e como aplicar.
G:\...\AutoPost\🧠 Cerebro\APRENDIZADOS.md → seção "Operações"
3
🤖 MEMORY.md do agente é atualizado
O agente específico (ex: @dev, @security) atualiza seu MEMORY.md no repo AIOX — ficando disponível para próximas sessões daquele agente no mesmo projeto.
.aios-core/development/agents/dev/MEMORY.md
4
📅 Motor de Promoção Mensal
Na primeira segunda-feira de cada mês, um script automático lê todos os cérebros locais de todos os projetos ativos e extrai padrões que aparecem em 2+ projetos.
Script: init-cerebro-projeto.ps1 · Destino: G:\Meu Drive\OBSidian\🌐 Global\PADROES_GLOBAIS.md
5
🌐 PADROES_GLOBAIS.md atualizado
O padrão vira conhecimento cross-projeto: "em projetos com Celery + asyncio, sempre usar NullPool". Agora está disponível para qualquer projeto futuro.
G:\Meu Drive\OBSidian\🌐 Global\PADROES_GLOBAIS.md
6
🚀 Novo Projeto começa mais inteligente
Ao iniciar um novo projeto, o script init-cerebro-projeto.ps1 pré-popula o cérebro local com os padrões globais. O agente @architect já sabe dos trade-offs antes de fazer qualquer decisão.
Resultado: menos erros repetidos, stack decisions mais rápidas

📱 E sobre trabalhar do celular?

Desde que o AIOX está no git (mathfonta/equipe-aiox), você pode:

  • Abrir o Claude.ai/code no browser em qualquer dispositivo
  • Fazer checkout do repo AIOX para um ambiente cloud (GitHub Codespaces)
  • Usar a extensão do VSCode Mobile com SSH para o seu PC
  • Ver o progresso das stories via GitHub (arquivos .md são legíveis no GitHub)
  • Adicionar ideias ao Obsidian Mobile → sincroniza via Google Drive → disponível na próxima sessão
⚠️ Limitação atual: Para implementar código (chamar @dev), você precisa de um ambiente com o Claude Code CLI ou extensão VSCode instalada. O celular funciona bem para revisar, planejar e adicionar ideias — mas não para implementação direta.
