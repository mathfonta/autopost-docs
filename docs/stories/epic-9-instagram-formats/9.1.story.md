## Descrição

O sistema atual usa 5 tipos de conteúdo temáticos (`post_simples`, `obra_andamento`, `obra_concluida`, `engajamento`, `bastidores`) herdados da Espectra — um cliente específico do segmento de construção civil. O AutoPost está evoluindo para um app genérico, agnóstico de nicho.

Esta story substitui os tipos temáticos pelos **4 formatos nativos do Instagram** (`feed_photo`, `carousel`, `reels`, `story`) e introduz um novo passo no fluxo de criação: a seleção de **sub-estratégia** — a intenção de conteúdo dentro de cada formato (ex: "Antes & Depois", "Tutorial POV", "Enquete").

A Fase 1 é conservadora: o upload continua sendo de imagem única para todos os formatos, a geração de copy permanece genérica. O objetivo é estabelecer a estrutura de dados e a UI sem quebrar o funcionamento atual.

## Decisões Técnicas

- IDs novos: `feed_photo`, `carousel`, `reels`, `story` — snake_case para manter padrão do backend
- `strategy` é nullable no banco — posts sem estratégia (antigos + Fase 1) continuam válidos
- Sub-estratégia é selecionada ANTES do upload, não no ContextModal (simplifica o fluxo)
- `POST_TYPE_MAP` continua existindo para lookup por ID — apenas com novos IDs
- Tipos antigos (`post_simples` etc.) podem aparecer em posts históricos — `POST_TYPE_MAP[id] ?? null` trata isso com fallback para `null`
- Cores novas por formato: Feed=Azul, Carousel=Âmbar, Reels=Vermelho, Story=Roxo

## Dev Notes

### Estrutura de dados das sub-estratégias (`lib/strategies.ts`)

```typescript
export interface Strategy {
  id:        string
  label:     string
  desc:      string        // 1 linha, ex: "Mostrar resultado entregue"
  objective: string        // alcance | engajamento | saves | conversão | confiança
  icon:      string        // emoji simples
}

export const STRATEGIES: Record<PostTypeId, Strategy[]> = {
  feed_photo: [...],
  carousel:   [...],
  reels:      [...],
  story:      [...],
}
```

### Novo fluxo de telas em `dashboard/page.tsx`

```
Screen: "dashboard" → "strategy" → "upload" → "preview" → (ContextModal)
```

- `"strategy"` recebe `postTypeId` e exibe `SubStrategySelector`
- Ao confirmar, seta `strategyId` e avança para `"upload"`
- `strategyId` é enviado no FormData junto com `content_type`

### Backend

- Migration: `ALTER TABLE content_requests ADD COLUMN strategy VARCHAR(50) NULL`
- Endpoint `POST /content-requests`: aceita `strategy` como Form field opcional
- Model `ContentRequest`: campo `strategy: Optional[str] = None`
