## Descrição

Com a Story 9.1, o usuário agora seleciona Formato + Sub-estratégia antes do upload. Porém, o Agente Copywriter continua ignorando essa escolha — gera copy genérica para todos os casos.

Esta story implementa a **Skill Library**: um dicionário de 23 instruções de prompt — uma por combinação de formato+estratégia — que é injetado no Agente Copywriter. A copy gerada passa a ter Hook, Corpo e CTA calibrados para a intenção exata declarada pelo usuário.

Exemplo: "Carrossel + Antes & Depois" gera hook de transformação com narrativa progressiva. "Reels + Hook de Choque" gera frase bombástica nos primeiros 3s + estrutura de revelação.

## Decisões Técnicas

- Chave do dicionário: `"{format}__{strategy_id}"` (double underscore) — ex: `"feed_photo__prova_social"`
- Injeção: campo `ESTRATÉGIA DO POST` no user_message, antes de `INTENÇÃO DO CLIENTE`
- Prioridade: `strategy` tem precedência sobre `content_type` legado quando ambos presentes
- Fallback: se `strategy` não encontrado no dict, comportamento padrão sem instrução adicional
- Retrocompatibilidade: posts sem `strategy` (antigos) continuam funcionando normalmente

### Estrutura da Skill Library

```python
STRATEGY_PROMPTS: dict[str, str] = {
    # Feed Photo (5)
    "feed_photo__prova_social": "...",
    "feed_photo__ancora_de_marca": "...",
    # ...

    # Carousel (6)
    "carousel__antes_depois": "...",
    # ...

    # Reels (6)
    "reels__hook_choque": "...",
    # ...

    # Story (6)
    "story__cta_link": "...",
    # ...
}
```

### Injeção no user_message

```python
strategy_section = ""
if strategy:
    key = f"{content_type}__{strategy}" if content_type else strategy
    prompt_instruction = STRATEGY_PROMPTS.get(key, "")
    if prompt_instruction:
        strategy_section = f"\n\nESTRATÉGIA DO POST: {prompt_instruction}"
```

### Pipeline

Em `_get_request_with_client`, adicionar:
```python
"strategy": req.strategy,
```

Em ambas as tasks (`generate_copy` e `retry_generate_copy`), adicionar:
```python
strategy=req.get("strategy"),
```
