## Dev Notes

### pip-audit vs safety
- **pip-audit** (recomendado): mantido pela PyPA, integra com pip, gratuito, CLI simples
- **safety**: requer token para acesso completo ao banco de dados
- **Decisão:** usar `pip-audit` como primário, `safety` como fallback opcional

### Instalação pip-audit
```bash
pip install pip-audit
pip-audit --requirement requirements.txt --format json
```

### GitHub Actions — backend (Python)
```yaml
name: Security Audit
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
jobs:
  dep-audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install pip-audit
      - run: pip-audit --requirement requirements.txt --fail-on CRITICAL
```

### Detecção de contexto em security-scan.md
```javascript
function detectProjectType(projectRoot) {
  if (fs.existsSync(path.join(projectRoot, 'requirements.txt'))) return 'python';
  if (fs.existsSync(path.join(projectRoot, 'pyproject.toml'))) return 'python';
  if (fs.existsSync(path.join(projectRoot, 'package.json'))) return 'node';
  return 'unknown';
}
```
