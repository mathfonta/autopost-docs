## Contexto

A Story 16.1 adicionou o parâmetro `marketing_intent` ao `copywriter.py` e os prompts de contexto. A Story 16.2 coleta a intenção no frontend. Esta story fecha o ciclo: propaga a intenção selecionada pelo usuário até o agente de copy, passando pelo endpoint REST e pelo modelo de dados.

---

## Descrição

### 1. Schema de ContentRequest (`POST /content-requests`)

Adicionar campo opcional `marketing_intent: str | None` ao endpoint de criação de conteúdo:

```python
# backend/app/schemas/content.py
class ContentRequestCreate(BaseModel):
    content_type:     str
    strategy:         str | None = None
    user_context:     str | None = None
    marketing_intent: str | None = None   # NOVO — opcional
```

### 2. Modelo ContentRequest

Adicionar coluna `marketing_intent VARCHAR(50) NULLABLE` à tabela `content_requests`:

```sql
-- Migration Alembic
ALTER TABLE content_requests
  ADD COLUMN marketing_intent VARCHAR(50) NULL;
```

### 3. Pipeline Celery (`pipeline.py`)

Passar `marketing_intent` do modelo para `generate_copy()`:

```python
copy_result = await generate_copy(
    photo_analysis=analysis,
    brand_profile=client.brand_profile,
    content_type=request.content_type,
    strategy=request.strategy,
    user_context=request.user_context,
    marketing_intent=request.marketing_intent,   # NOVO
)
```

### 4. Frontend: `api.ts`

Adicionar `marketing_intent` ao payload do `POST /content-requests`:

```typescript
formData.append("marketing_intent", selectedIntent ?? "");
```
