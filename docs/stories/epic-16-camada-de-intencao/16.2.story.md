## Contexto

Com o mapeamento de intenções disponível (`intents.ts`), esta story cria a tela `IntentSelector` — a nova entrada do fluxo de criação de conteúdo. O usuário vê 5 objetivos de negócio e escolhe o que representa sua intenção com o post. O sistema navega automaticamente para o formato e estratégia recomendados.

O fluxo atual começa no `ContentTypeBar` (seleção de formato). O novo fluxo insere o `IntentSelector` antes dele, mas preserva um caminho de escape para o fluxo manual.

---

## Descrição

### Novo componente: `IntentSelector`

Tela full-page (mesmo padrão de `SubStrategySelector`) exibindo os 5 intents como cards verticais:

```
┌─────────────────────────────────────────────┐
│ ← [AutoPost]                                │
│                                             │
│  O que você quer com este post?             │
│                                             │
│ ┌─────────────────────────────────────────┐ │
│ │ 💰  Gerar orçamentos               [→] │ │
│ │     Transformar visitantes em contatos  │ │
│ └─────────────────────────────────────────┘ │
│ ┌─────────────────────────────────────────┐ │
│ │ 📈  Ganhar seguidores              [→] │ │
│ │     Aumentar o alcance do perfil        │ │
│ └─────────────────────────────────────────┘ │
│   ... (5 cards no total)                    │
│                                             │
│     ─────────────────────────               │
│     ou escolher formato manualmente →       │
└─────────────────────────────────────────────┘
```

**Ao selecionar um intent:**
1. Salva o intent no estado do `DashboardPage` (`selectedIntent`)
2. Navega para `ContentTypeBar` com o `postTypeId` recomendado pré-selecionado
3. No `SubStrategySelector`, o `strategyId` recomendado aparece no topo com badge "Recomendado"

**Ao clicar "escolher manualmente":**
- Navega direto para `ContentTypeBar` sem `selectedIntent` — fluxo atual intacto

### Alterações no `DashboardPage`

- Adicionar estado `selectedIntent: MarketingIntent | null`
- Adicionar estado `screen: "intent"` à union existente
- Lógica: ao abrir novo post (`handleNewPostAction`), ir para `"intent"` em vez de `"upload"`
- Passar `selectedIntent` adiante pelo fluxo até chegar ao `SubStrategySelector`

### Alterações no `SubStrategySelector`

- Receber prop opcional `recommendedStrategyId?: string`
- Quando presente, renderizar o card recomendado primeiro na lista com badge "Recomendado para seu objetivo"

## Riscos

- Adicionar uma tela no início do fluxo pode aumentar o atrito para usuários experientes → mitigação: o link "escolher manualmente" pula direto para o fluxo atual; considerar no futuro um toggle de preferência
