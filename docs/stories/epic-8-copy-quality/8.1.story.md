# Story 8.1 — Janela de Contexto para Geração de Copy

**Epic:** 8 — Qualidade e Personalização da Copy
**Status:** InProgress
**Agente executor:** @dev (YOLO mode)
**Data:** 2026-04-26

---

## Descrição

Atualmente o usuário seleciona a foto e o pipeline dispara imediatamente, sem qualquer informação adicional além da análise visual da IA. Para segmentos como construção civil, onde o mesmo revestimento pode ser porcelanato importado ou cerâmica simples, o contexto do usuário é decisivo para a qualidade da copy.

Esta story adiciona uma **Janela de Contexto opcional** entre a seleção da foto e o disparo do pipeline. O usuário pode descrever brevemente o serviço, material ou produto fotografado. O Agente Analista e o Copywriter usam esse contexto como dado adicional — mantendo a análise visual como fonte primária (80–90%) e o contexto como especificador.

---

## Acceptance Criteria

- [x] AC1: Após selecionar a foto, antes do POST, exibe modal de contexto com campo de texto opcional (máx. 200 chars)
- [x] AC2: Modal exibe preview da foto selecionada
- [x] AC3: Botão "Gerar copy" e link "Pular" ambos funcionais
- [x] AC4: Contexto é enviado no POST `/content-requests` como campo `user_context`
- [x] AC5: Campo `user_context TEXT NULL` existe na tabela `content_requests`
- [x] AC6: Agente Analista recebe contexto e o inclui no user message para o Claude
- [x] AC7: Agente Copywriter recebe contexto e o injeta como `#CONTEXTO_DO_USUARIO` no prompt
- [x] AC8: Pipeline sem contexto (campo vazio/null) funciona exatamente como antes

---

## Escopo

**IN:**
- Modal de contexto no frontend (PhotoUploadFAB)
- Campo `user_context` no model, migration, endpoint, schema
- Injeção de contexto nos prompts do Analista e Copywriter
- Passagem do contexto pelo pipeline Celery

**OUT:**
- Redesign do fluxo de upload (mantém fluxo existente)
- Exibição do contexto na tela de aprovação (backlog)
- Contexto no retry de copy (passa o original — fora do escopo v1)

---

## Decisões Técnicas (Decision Log)

- Contexto injetado em AMBOS os agentes (Analista + Copywriter) para enriquecer a análise e a copy
- Campo nullable — backward compatible com posts sem contexto
- Limite de 200 chars no frontend (campo texto simples, não multiline)
- Preview da foto no modal: `URL.createObjectURL(file)` — sem upload adicional
