# Story 8.3 — Copy em Múltiplos Formatos

**Epic:** 8 — Qualidade e Personalização da Copy
**Status:** Done
**Agente executor:** @dev
**Data criação:** 2026-04-27

---

## Descrição

Atualmente o Copywriter gera uma única legenda para o post. Para criadores de conteúdo do segmento de construção/reforma, a mesma foto pode ser usada em formatos diferentes: uma legenda longa para o feed (engajamento + storytelling), uma legenda curta para agilidade, e um texto para Stories (tom mais conversacional e imediato).

Esta story faz o Copywriter gerar **3 variações de copy** em uma única chamada. Na tela de aprovação, o usuário escolhe qual usar antes de aprovar — sem precisar fazer retry ou editar manualmente.

---

## Acceptance Criteria

- [ ] AC1: Agente Copywriter gera 3 variações em uma chamada: `caption_long` (até 400 chars), `caption_short` (até 150 chars), `caption_stories` (até 100 chars, tom conversacional)
- [ ] AC2: As 3 variações são armazenadas em colunas separadas na tabela `content_requests`
- [ ] AC3: Migration criada com as 3 novas colunas — backward compatible (nullable, posts antigos preservados)
- [ ] AC4: A variação selecionada pelo usuário é armazenada em `caption_selected` (enum: `long` | `short` | `stories`) e é a que vai para publicação
- [ ] AC5: Na tela de aprovação, exibe seletor visual com as 3 variações — usuário clica para selecionar antes de aprovar
- [ ] AC6: Default de seleção: `long` (comportamento atual preservado se usuário aprovar sem selecionar)
- [ ] AC7: Posts antigos (sem as variações) continuam funcionando — exibem apenas o `caption` existente sem o seletor
- [ ] AC8: O campo `caption` principal é populado com a variação selecionada no momento da publicação

## Escopo

**IN:**
- Agente Copywriter gera 3 variações em JSON estruturado
- Migration: `caption_long`, `caption_short`, `caption_stories`, `caption_selected`
- Seletor de variação na tela de aprovação (PostCard / página de detalhe)

- Geração de variações no retry (usa as variações já geradas — fora do escopo v1)
- Variação customizada (texto livre além das 3 opções)
- Exibição das 3 variações no histórico
- Variações para carrossel/multi-foto (escopo futuro)

## Dev Notes

### Contexto das Stories anteriores
- Story 8.1 estabeleceu o padrão de campos extras no `content_request` e pipeline
- O `caption` atual em `content_requests` é a legenda publicada — manter esse campo como destino final
- O Copywriter retorna atualmente um JSON com `caption` e `hashtags` — estender para retornar 3 variações

### Backend — Estrutura de Saída do Copywriter

**Formato atual:**
```json
{ "caption": "...", "hashtags": ["#tag1"] }
```

**Novo formato:**
```json
{
  "caption_long": "Legenda completa com storytelling... até 400 chars",
  "caption_short": "Versão direta. Até 150 chars.",
  "caption_stories": "Ei, olha isso! 👀 Até 100 chars",
  "hashtags": ["#construção", "#reforma"]
}
```

O campo `caption` principal (para publicação) continua sendo populado — com o valor de `caption_long` por default, ou com a variação selecionada pelo usuário.

### Backend — Migration

```sql
ALTER TABLE content_requests
  ADD COLUMN IF NOT EXISTS caption_long TEXT,
  ADD COLUMN IF NOT EXISTS caption_short VARCHAR(150),
  ADD COLUMN IF NOT EXISTS caption_stories VARCHAR(100),
  ADD COLUMN IF NOT EXISTS caption_selected VARCHAR(10) DEFAULT 'long'
    CHECK (caption_selected IN ('long', 'short', 'stories'));
```

### Backend — Prompt do Copywriter

Adicionar instrução para retornar JSON com as 3 variações. Exemplo de instrução:

```
Gere 3 variações de legenda para Instagram:
1. caption_long: legenda completa com storytelling (máx 400 chars)
2. caption_short: versão objetiva e direta (máx 150 chars)  
3. caption_stories: texto para Stories, tom conversacional (máx 100 chars)
Retorne JSON: {"caption_long": "...", "caption_short": "...", "caption_stories": "...", "hashtags": [...]}
```

### Backend — Pipeline

Em `pipeline.py`, após `generate_copy`:
- Salvar as 3 variações nas colunas correspondentes
- Popular `caption` com `caption_long` por default
- Popular `caption_selected` com `'long'` por default

### Frontend — Seletor de Variação

Na tela de aprovação (`PostCard.tsx` ou `app/posts/[id]/page.tsx`):
- Se `caption_long` existe → exibir seletor com 3 tabs/botões: "Completa" | "Curta" | "Stories"
- Ao selecionar: `PATCH /content-requests/{id}` com `{ caption_selected: 'short', caption: caption_short }`
- Se `caption_long` não existe (post antigo) → exibir apenas o `caption` atual sem seletor

### API — Endpoint de Seleção

Verificar se `PATCH /content-requests/{id}` já existe para edição de caption (CaptionEditor da Story 5.1). Se sim, estender para aceitar `caption_selected`. Se não, criar.
