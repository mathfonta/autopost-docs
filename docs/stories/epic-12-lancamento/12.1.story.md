## Story
**As a** novo usuário que acabou de conectar o Instagram,
**I want** ser guiado imediatamente para criar meu primeiro post ao terminar o onboarding,
**so that** eu veja um resultado real publicado no meu perfil em menos de 2 minutos, sem precisar entender o dashboard primeiro.

## Acceptance Criteria

1. **AC1 — CTA primário na StepComplete:** O botão principal da tela de conclusão do onboarding deve ser "Criar meu primeiro post →" (não "Ir para o Dashboard").
2. **AC2 — Redirecionamento com intenção:** Ao clicar no CTA primário, o usuário é redirecionado para `/dashboard?action=new-post`.
3. **AC3 — UploadScreen auto-abre:** Quando o dashboard detecta `?action=new-post` na URL, o `UploadScreen` abre automaticamente, sem interação adicional do usuário.
4. **AC4 — Limpeza da URL:** Após o `UploadScreen` abrir, o parâmetro `action=new-post` é removido da URL (sem reload) para evitar re-trigger ao recarregar.
5. **AC5 — CTA secundário preservado:** Deve existir um link secundário discreto "Explorar o dashboard primeiro" que navega para `/dashboard` sem o parâmetro.
6. **AC6 — Push notification mantida:** O comportamento existente de solicitar permissão de push notification é preservado exatamente como está.
7. **AC7 — Sem regressão no fluxo existente:** Usuários que já fizeram onboarding e acessam `/dashboard` diretamente não são afetados.

### Estado existente no dashboard (não recriar)
O `dashboard/page.tsx` usa uma **máquina de estados Screen**, NÃO um boolean `showUpload`:
```tsx
// Já existe no dashboard — máquina de estados
type Screen = "dashboard" | "strategy" | "upload" | "preview" | "generating" | "approval";
const [screen, setScreen] = useState<Screen>("dashboard");
const [postTypeId, setPostTypeId] = useState<PostTypeId | null>(null);

// A tela de upload é renderizada condicionalmente via screen === "upload"
// e requer que postTypeId esteja definido
```
Para auto-abrir o upload via searchParam, chamar:
```tsx
setPostTypeId('photo'); // tipo padrão — usuário pode trocar na tela
setScreen('upload');
```
**NUNCA** procurar `showUpload` ou `setShowUpload` — esses estados não existem.

### Cuidado com useEffect e searchParams
`useSearchParams()` em Next.js App Router deve ser usado dentro de `<Suspense>`. Verificar se o componente raiz do dashboard já está envolvido com Suspense antes de adicionar a hook.

Se `dashboard/page.tsx` não usa `useSearchParams` ainda, adicioná-lo requer envolver o componente com `<Suspense>` ou usar `useSearchParams` em um Client Component filho separado.

### Pattern recomendado para limpeza de URL
```tsx
const searchParams = useSearchParams();
const router = useRouter();

useEffect(() => {
  if (searchParams.get('action') === 'new-post') {
    setPostTypeId('photo'); // abre upload com tipo foto como padrão
    setScreen('upload');
    router.replace('/dashboard'); // limpa sem reload
  }
}, []); // sem dependências — só roda na montagem
```
