## User Story

**As a** produto AutoPost,  
**I want** uma camada de busca Exa isolada, testável e com fallback gracioso,  
**so that** qualquer agente do pipeline possa enriquecer conteúdo com dados externos sem risco de quebrar o fluxo principal.

---

## Descrição

Cria a camada base para usar o Exa Search API no pipeline:
função isolada em `app/tools/exa_search.py` (mesmo padrão de `transcription.py`),
configuração plugável via `EXA_PROVIDER`, cache Redis de 24h e graceful fallback.

Esta story não muda nada visível para o usuário.

---

## Acceptance Criteria

- [ ] AC1: `EXA_PROVIDER: str = "disabled"` e `EXA_API_KEY: str = ""` declarados em `config.py` — mesmo padrão plugável de `COPY_PROVIDER` e `TRANSCRIPTION_PROVIDER`
- [ ] AC2: `app/tools/exa_search.py` criado com:
  - `async def search_exa_trends(segment: str, content_type: str, days_back: int = 30) -> str | None`
  - Usa `httpx.AsyncClient` com timeout de 10s — **não** o MCP server node
  - Retorna string formatada com `[CONTEXTO DE MERCADO RECENTE]` ou `None`
  - Usa `highlights` mode (não full text) — economiza tokens
  - Filtra por `startPublishedDate` (últimos N dias) — evita conteúdo velho
- [ ] AC3: Cache Redis com chave `exa:trends:{segment}:{content_type}` — TTL 86400s (24h). Retorna cache hit sem chamar a API
- [ ] AC4: Graceful fallback obrigatório: qualquer exceção (timeout, 4xx, 5xx, network error) loga `WARNING` e retorna `None` — **nunca** propaga exceção para o pipeline
- [ ] AC5: Se `EXA_PROVIDER != "exa"`: retorna `None` imediatamente sem chamar a API (zero overhead)
- [ ] AC6: `EXA_API_KEY` e `EXA_PROVIDER=disabled` documentados no `backend/.env.example`
- [ ] AC7: `tests/test_exa_search.py` com mocks cobrindo: sucesso, timeout, fallback (API down), cache hit, provider desativado

## Implementação de referência

```python
# app/tools/exa_search.py
import httpx
from datetime import datetime, timedelta
from app.core.config import settings
from app.core.logging import logger

EXA_BASE_URL = "https://api.exa.ai"

# Queries semânticas por content_type (mesmo dict style dos _VIRAL_TRIGGERS)
_CONTENT_TYPE_QUERIES = {
    "obra_realizada":      "acabamento construção civil residencial tendência qualidade",
    "obra_em_andamento":   "construção civil Brasil 2025 materiais inovação canteiro",
    "antes_depois":        "reforma residencial tendência design interiores transformação",
    "engajamento":         "mercado imobiliário construção civil notícias destaque",
    "bastidores":          "gestão obra equipe construção profissional",
}

async def search_exa_trends(
    segment: str,
    content_type: str,
    days_back: int = 30
) -> str | None:
    """
    Busca tendências recentes no Exa. Retorna bloco formatado para injeção
    no prompt do copywriter, ou None se Exa estiver desativado/falhar.
    """
    if settings.EXA_PROVIDER != "exa":
        return None

    # Cache Redis
    cache_key = f"exa:trends:{segment}:{content_type}"
    # [verificar cache antes de chamar API — ver implementação completa]

    try:
        start_date = (datetime.now() - timedelta(days=days_back)).strftime("%Y-%m-%dT00:00:00Z")
        query = _build_query(segment, content_type)

        async with httpx.AsyncClient(timeout=10.0) as client:
            response = await client.post(
                f"{EXA_BASE_URL}/search",
                headers={
                    "x-api-key": settings.EXA_API_KEY,
                    "Content-Type": "application/json"
                },
                json={
                    "query": query,
                    "numResults": 5,
                    "startPublishedDate": start_date,
                    "contents": {
                        "highlights": {
                            "numSentences": 3,
                            "highlightsPerUrl": 2
                        },
                        "text": False  # só highlights, economiza tokens
                    },
                    "category": "news"
                }
            )
            response.raise_for_status()

        data = response.json()
        result = _format_for_prompt(data.get("results", []))
        # [salvar no cache Redis com TTL 86400]
        return result

    except Exception as e:
        logger.warning(f"[exa] falhou — continuando sem tendências: {e}")
        return None


def _build_query(segment: str, content_type: str) -> str:
    base = _CONTENT_TYPE_QUERIES.get(content_type, "construção civil tendências")
    return f"{base} {segment} site:br"


def _format_for_prompt(results: list) -> str | None:
    snippets = []
    for r in results[:3]:
        highlights = r.get("highlights", [])
        if highlights:
            snippets.append(" ".join(highlights[:2]))

    if not snippets:
        return None

    lines = ["[CONTEXTO DE MERCADO RECENTE]"]
    for s in snippets:
        lines.append(f"• {s}")
    return "\n".join(lines)
```
