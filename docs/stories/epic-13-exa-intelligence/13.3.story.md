## User Story

**As a** usuário do AutoPost,  
**I want** que os hashtags gerados reflitam o que está em alta nesta semana no meu nicho,  
**so that** meus posts tenham mais alcance orgânico sem eu precisar pesquisar hashtags manualmente.

---

## Descrição

Hoje as hashtags são geradas pelo LLM com base na lista `_VIRAL_TRIGGERS` (hardcoded por segmento)
e no prompt geral. O resultado é previsível e não reflete o que está em alta **hoje**.

Esta story usa os dados Exa já buscados na Story 13.2 para **refinar o bloco de hashtags**
com base em termos que estão com volume real na semana.

Adicionalmente, cria uma busca específica de hashtags se `exa_trends_context` indicar
termos novos não cobertos pela lista estática.

---

## Acceptance Criteria

- [ ] AC1: `generate_copy_with_ai()` extrai hashtags candidatas do `exa_trends_context` (regex simples: `#\w+`)
- [ ] AC2: Hashtags Exa são injetadas no prompt como sugestão de prioridade: `#HASHTAGS_EM_ALTA: {lista}`
- [ ] AC3: O LLM é instruído a priorizar hashtags Exa + complementar com seu próprio conhecimento para chegar a 20-25 tags
- [ ] AC4: Hashtags retornadas no JSON (`hashtags: list[str]`) devem incluir ao menos 3 hashtags oriundas do Exa quando disponíveis
- [ ] AC5: Sem Exa (provider desativado ou falha): comportamento atual preservado — hashtags 100% geradas pelo LLM
- [ ] AC6: Log `[copywriter] hashtags exa aproveitadas: {N}/{total}` para monitorar qualidade
