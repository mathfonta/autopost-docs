# Story 2.1 — Agente Analista de Foto (Claude Haiku)

**Status:** Done  
**Epic:** 2 — Agentes de IA  
**Story:** 2.1  
**Estimativa:** 3 pontos  
**Depende de:** 1.4

---

## Descrição

Como AutoPost, preciso que o Agente Analista receba uma foto e use Claude Haiku para analisar qualidade, identificar o tipo de conteúdo (obra realizada, antes/depois, dica, etc.) e decidir se a foto é boa o suficiente para ser publicada — substituindo o stub atual em `analyze_photo`.

---

## Critérios de Aceitação

- [x] AC1: Task `analyze_photo` chama Claude Haiku com a imagem (via URL do R2) e retorna análise estruturada
- [x] AC2: `analysis_result` contém: `quality` (good/acceptable/bad), `content_type` (obra_realizada/antes_depois/dica/promocao), `description` (texto livre), `publish_clean` (bool)
- [x] AC3: Foto com `quality=bad` → status = `failed`, cliente notificado via campo `error_message` com sugestão
- [x] AC4: Prompt do analista usa `brand_profile` do cliente (segmento, tom de voz) para contextualizar análise
- [x] AC5: Custo máximo por análise: ≤ 500 tokens de entrada + 200 saída (Claude Haiku) — MAX_TOKENS=300
- [x] AC6: Timeout de 30s — se Claude não responder, task entra em retry
- [x] AC7: Testes com mock da API Anthropic (não chama API real nos testes)

## Dev Notes

- Claude Haiku: modelo `claude-haiku-4-5-20251001` — mais barato para análise de imagem
- Anthropic SDK suporta visão: passar `{"type": "image", "source": {"type": "url", "url": photo_url}}`
- `brand_profile` vem do `ContentRequest.client_id` → join com `Client.brand_profile`
- Prompt deve solicitar resposta em JSON válido (usar `json.loads` com try/except)
- asyncio.run() dentro da task Celery — padrão já estabelecido na Story 1.4
