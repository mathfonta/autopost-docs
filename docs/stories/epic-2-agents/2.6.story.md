# Story 2.6 — API de Conteúdo (Upload, Histórico, Aprovação)

**Status:** Done  
**Epic:** 2 — Agentes de IA  
**Story:** 2.6  
**Estimativa:** 3 pontos  
**Depende de:** 2.3, 2.4

---

## Descrição

Como cliente do AutoPost, quero submeter fotos via API, acompanhar o histórico de conteúdos gerados e aprovar ou rejeitar o copy/design antes da publicação — para ter controle total sobre o que é publicado no meu Instagram.

---

## Critérios de Aceitação

- [x] AC1: `POST /content-requests` recebe upload de foto, faz upload para R2 e dispara o pipeline Celery
- [x] AC2: `GET /content-requests` retorna histórico paginado do cliente (apenas os próprios registros — tenant isolation)
- [x] AC3: `GET /content-requests/{id}` retorna detalhes completos de um request (analysis, copy, design, publish results)
- [x] AC4: `POST /content-requests/{id}/approve` transiciona status para `approved` e dispara `publish_post`
- [x] AC5: `POST /content-requests/{id}/reject` transiciona status para `rejected` com motivo opcional
- [x] AC6: Schemas de resposta tipados via Pydantic com todos os campos de resultado

---

## Tasks

- [x] Criar `app/schemas/content.py` — schemas Pydantic para request e response

## Escopo

**IN:** Upload de foto, histórico paginado, aprovação/rejeição de copy+design  
**OUT:** WebSocket para notificações em tempo real (Epic 3), agendamento de publicação (Epic 3)
