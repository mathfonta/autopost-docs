## Contexto

Para que o sistema possa sugerir temas de carrossel sem upload, precisa de uma biblioteca de temas organizados por nicho. Esta story cria essa biblioteca como configuração estática (arquivo Python) e expõe um endpoint REST para o frontend listá-los.

### 2. Endpoint REST: `GET /insights/themes`

```python
@router.get("/themes")
async def list_themes(
    current_client: Client = Depends(get_current_client),
):
    """
    Retorna temas de carrossel disponíveis para o nicho do client autenticado.
    Fallback para 'default' se o nicho não tiver temas específicos.
    """
```

Lógica de nicho: buscar `current_client.brand_profile.get("segment", "")`, normalizar para lowercase, fazer matching com as chaves de `THEME_LIBRARY`. Se não encontrar correspondência, retornar `default` + temas gerais.

**Resposta:**
```json
{
  "segment": "construção civil",
  "themes": [
    {
      "id": "cc_erros_reforma",
      "title": "5 erros que encarecem qualquer reforma",

      "objective": "autoridade",
      "slide_count": 7,
      "tags": ["reforma", "dicas", "planejamento"]
    }
  ]
}
```

## Acceptance Criteria

- [ ] AC1: `backend/app/data/theme_library.py` criado com pelo menos 3 nichos + default, mínimo 10 temas total
- [ ] AC2: `GET /insights/themes` requer autenticação e retorna temas do nicho do client autenticado
- [ ] AC3: Nicho sem temas específicos recebe temas `default` + temas gerais mesclados
- [ ] AC4: Matching de nicho é case-insensitive e tolera variações (ex: "Construção Civil" = "construção civil")
- [ ] AC5: Endpoint documentado no OpenAPI (`/docs`)
- [ ] AC6: Testes unitários cobrem: nicho com temas, nicho sem temas (fallback), client sem brand_profile

---

## Scope

### IN
- `backend/app/data/theme_library.py` — biblioteca estática
- `GET /insights/themes` — novo endpoint no router de insights

### OUT
- Não armazenar temas em banco de dados (arquivo estático nesta story — persistência em banco é evolução futura)
- Não permitir temas customizados pelo usuário nesta versão

---

## Dependências

- `backend/app/api/insights.py` — router onde o endpoint será adicionado
- `backend/app/core/auth.py` — dependência de autenticação
- `backend/app/models/client.py` — `brand_profile` com `segment`

## Riscos

- Nicho do `brand_profile` pode ter variações ortográficas (ex: "construção" vs "construcao") → mitigação: normalizar com `unidecode` ou comparação simples sem acentos
- Biblioteca de temas crescer sem estrutura → mitigação: arquivo separado por categoria em futuras versões
