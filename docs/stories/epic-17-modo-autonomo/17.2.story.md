## Contexto

O pipeline atual (`pipeline.py`) sempre começa com uma foto enviada pelo usuário. Esta story cria um pipeline alternativo para `content_type = autonomous_carousel` — sem upload, sem análise de imagem. O conteúdo parte do tema selecionado e do `brand_profile` do cliente.

Esta story contém uma **decisão arquitetural crítica que requer aprovação antes da implementação** (ver seção Decisão Pendente).

---

## Decisão Pendente — REQUER APROVAÇÃO DO USUÁRIO

O slide designer precisa gerar imagens de card com texto sobre fundo colorido. Há 3 opções:

### Opção A — Pillow (Python, sem custo adicional)
- Gera imagens PNG com texto + fundo usando a cor primária do `brand_profile`
- Controle total, zero dependência externa, sem custo de API
- Limitação: tipografia e layout básicos
- **Recomendação @architect: esta opção**

### Opção B — HTML template → screenshot (Playwright)
- Renderiza template HTML com Tailwind, tira screenshot
- Layout rico, fontes do sistema, bordas e sombras
- Adiciona dependência Playwright no Railway; aumenta memory footprint (~200MB)
- Latência: ~3s por card (headless browser)

### Opção C — Geração por API de imagem (Ideogram, DALL-E)
- Imagens de alta qualidade com IA
- Custo por imagem (~$0.04–0.08 por card = $0.28–0.56 por carrossel de 7 cards)
- Menos controle sobre texto exato e layout da marca

**Aguardando decisão antes de implementar.**

```python
                ...

        }

```

### 2. Novo agente: `backend/app/agents/slide_designer.py`

Implementação conforme decisão da Opção A/B/C.

Para **Opção A (Pillow)**:
```python
async def render_slide_image(
    slide: dict,   # {"headline": "...", "body": "..."}
    brand_profile: dict,  # primary_color, company_name
    slide_index: int,
    total_slides: int,
) -> bytes:  # PNG bytes para upload no R2
```

### 3. Nova task Celery: `pipeline_autonomous.py`

```python
@celery_app.task
def process_autonomous_carousel(request_id: str):
    """
    Pipeline: theme → slide_structure → render_images → generate_caption → save
    """
    1. Carregar ContentRequest (type=autonomous_carousel, theme_id preenchido)
    2. Buscar tema em THEME_LIBRARY
    3. generate_slide_structure() → estrutura de slides
    4. Para cada slide: render_slide_image() → upload R2
    5. generate_copy() com estrutura de slides como contexto
    6. Salvar result e mudar status para awaiting_approval
```

### 4. Endpoint de criação: `POST /content-requests/autonomous`

```python
class AutonomousCarouselCreate(BaseModel):
    theme_id:         str
    marketing_intent: str | None = None
```

Alternativa: reutilizar `POST /content-requests` com `content_type=autonomous_carousel` e `theme_id` como campo extra.

## Riscos

| Risco | Probabilidade | Mitigação |
|-------|--------------|-----------|
| Claude gera estrutura de slides inconsistente | Média | Few-shot prompting com exemplos reais no system prompt |
| Pillow produz layout muito básico e rejeição do usuário | Alta | Validar exemplo de output com Matheus antes de implementar |
| Timeout de 120s excedido para carrossel de 10 slides | Baixa | Processar slides em paralelo (asyncio.gather) |
| R2 upload falha para um dos slides | Baixa | Retry com backoff; se falhar, marcar request como failed com mensagem clara |
