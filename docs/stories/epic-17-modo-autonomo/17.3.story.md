# Story 17.3 — Interface de Seleção de Tema (Frontend)

**Epic:** 17 — Modo Autônomo: Carrossel Temático
**Status:** Draft
**Agente executor:** @dev
**Data:** 2026-06-10
**Prioridade:** Média
**Depende de:** Story 17.1 (endpoint `/insights/themes`)

---

## Contexto

Com a biblioteca de temas disponível via API, esta story cria a interface onde o usuário seleciona um tema para gerar um carrossel autônomo. O ponto de entrada é um novo botão/card no `ContentTypeBar` — "Criar Carrossel Automático" — que abre a tela de seleção de temas.

---

## Descrição

### 1. Novo item no `ContentTypeBar`: "Automático"

Adicionar um item ao `ContentTypeBar` com destaque visual diferente dos outros (badge "NOVO" ou borda colorida):

```
[Feed Photo] [Carrossel] [Reels] [Story] [✨ Automático]
```

Ao clicar, navega para a nova tela `ThemeSelector` em vez do `SubStrategySelector`.

### 2. Novo componente: `ThemeSelector`

Tela full-page com lista de temas disponíveis para o nicho do usuário.

**Layout:**
```
← [AutoPost]
   Carrossel Automático

   Seu nicho: Construção Civil

   ┌──────────────────────────────────────────┐
   │ 🏗️  5 erros que encarecem qualquer reforma│
   │     Erros comuns de planejamento         │
   │     7 cards  ·  autoridade          [→]  │
   └──────────────────────────────────────────┘
   ┌──────────────────────────────────────────┐
   │ 📋  O que conferir antes de assinar...   │
   │     Checklist de contratação             │
   │     8 cards  ·  autoridade          [→]  │
   └──────────────────────────────────────────┘
   ... (demais temas)
```

- Busca temas via `GET /insights/themes` no mount
- Loading state: skeleton de 3 cards
- Erro: mensagem + botão retry

### 3. Fluxo ao selecionar tema

1. Usuário seleciona tema → POST `/content-requests/autonomous` com `theme_id`
2. Navega para `GeneratingScreen` (componente existente, reutilizado)
3. Quando pronto → `ApprovalScreen` (componente existente, reutilizado)
