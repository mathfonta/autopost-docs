## Descrição

Como cliente AutoPost, preciso de uma tela principal que mostre meus posts pendentes de aprovação
em cards visuais estilo Instagram — para que eu possa aprovar ou rejeitar com um toque, sem precisar
entender nada de tecnologia.

Esta é a tela mais importante do produto: 80% do uso diário acontece aqui.

---

## Critérios de Aceitação

- [ ] AC1: Dashboard (`/dashboard`) carrega lista de posts via `GET /content-requests` com polling a cada 5 segundos
- [ ] AC2: Posts com status `awaiting_approval` aparecem em destaque no topo com badge "Aguardando aprovação"
- [ ] AC3: Cada card de post exibe: imagem processada (1080×1080 em miniatura), legenda gerada, status atual com ícone colorido
- [ ] AC4: Botão "Aprovar" (verde) chama `POST /content-requests/{id}/approve` — desabilita durante loading, toast de sucesso
- [ ] AC5: Botão "Rejeitar" (vermelho) abre modal com campo de motivo opcional — chama `POST /content-requests/{id}/reject`
- [ ] AC6: Status do pipeline exibido em tempo real: Analisando → Gerando copy → Preparando design → Aguardando aprovação → Publicando → Publicado
- [ ] AC7: Posts publicados mostram link "Ver no Instagram" apontando para `publish_result.permalink`
- [ ] AC8: Tela vazia (sem posts) mostra call-to-action "Envie sua primeira foto"
- [ ] AC9: Botão flutuante "Nova Foto" → navega para tela de upload (placeholder — implementado em v1.1 via API)
- [ ] AC10: Responsivo — layout em coluna única no mobile, grid 2 colunas no desktop

## Dev Notes

### Contratos de API — shapes dos campos JSON

```typescript
// item.design_result — salvo pelo agente Designer
interface DesignResult {
  processed_photo_url: string;  // URL R2 da imagem 1080×1080 ← usar no card
  type: string;                 // ex: "border_glow"
  dimensions: "1080x1080";
  file_size_kb: number;
  r2_key: string;               // chave R2 para regenerar URL se expirar
}

// item.copy_result — salvo pelo agente Copywriter
interface CopyResult {
  caption: string;              // legenda completa ← exibir no card
  hashtags: string[];
  cta: string;
  suggested_time: string;       // ex: "18:00"
}

// item.publish_result — salvo pelo agente Publicador
interface PublishResult {
  instagram_post_id: string | null;
  facebook_post_id: string | null;
  permalink: string | null;     // ← URL "Ver no Instagram" (AC7)
  metrics?: Record<string, unknown>; // adicionado 24h depois
}

// Helpers de extração segura
const getProcessedImageUrl = (item: ContentRequest) =>
  item.design_result?.processed_photo_url ?? item.photo_url;

const getPermalink = (item: ContentRequest) =>
  item.publish_result?.permalink ?? null;

const getCaption = (item: ContentRequest) =>
  item.copy_result?.caption ?? "";
```

### Tratamento de imagem R2 expirada

`processed_photo_url` pode expirar (presigned URL). Usar fallback no card:

```tsx
<img
  src={getProcessedImageUrl(item)}
  onError={(e) => { (e.target as HTMLImageElement).src = item.photo_url; }}
  alt="Post"
/>
```

### Polling simples (sem WebSocket)

```typescript
// hooks/useContentRequests.ts
export function useContentRequests() {
  const [posts, setPosts] = useState<ContentRequest[]>([]);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    let active = true;
    const fetchPosts = async () => {
      if (document.visibilityState === "hidden") return;
      try {
        const { data } = await api.get<ContentRequestListResponse>('/content-requests');
        if (active) { setPosts(data.items); setError(null); }
      } catch {
        if (active) setError("Erro ao carregar posts");
      }
    };
    fetchPosts();
    const interval = setInterval(fetchPosts, 5000);
    return () => { active = false; clearInterval(interval); };
  }, []);

  return { posts, error };
}
```

## Riscos

- Imagem do R2 pode ser presigned URL expirada — backend deve retornar URL fresca ou usar URL pública
- Polling agressivo (5s) pode causar muitas chamadas com muitos posts — paginar se > 20 posts
- Rejeição sem motivo: backend aceita campo opcional, frontend não deve obrigar
