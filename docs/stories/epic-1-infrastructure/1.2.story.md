# Story 1.2 — FastAPI + Autenticação JWT (Supabase Auth)

**Status:** Done  
**Epic:** 1 — Infraestrutura Base  
**Story:** 1.2  
**Estimativa:** 3 pontos  
**Depende de:** 1.1

---

## Descrição

Como cliente do AutoPost, quero me registrar e fazer login na plataforma para que minha conta seja criada com segurança e eu receba tokens JWT para autenticar chamadas à API.

---

## Critérios de Aceitação

- [x] AC1: `POST /auth/register` cria usuário no Supabase Auth e registro `Client` no banco, retorna tokens JWT (201)
- [x] AC2: `POST /auth/register` retorna 409 se email já cadastrado
- [x] AC3: `POST /auth/login` autentica via Supabase Auth e retorna tokens JWT
- [x] AC4: `POST /auth/login` tem rate limiting de 10 req/min por IP
- [x] AC5: `POST /auth/refresh` renova access_token usando refresh_token
- [x] AC6: `GET /auth/me` retorna dados do client autenticado (requer Bearer token válido)
- [x] AC7: Senhas nunca são armazenadas no banco AutoPost (delegado ao Supabase Auth)
- [x] AC8: Token inválido em qualquer rota protegida retorna 401

## Dev Notes

- Import circular resolvido movendo `limiter` para `app/core/limiter.py`
- Alembic precisa de `connect_args={"statement_cache_size": 0}` no `async_engine_from_config`
- Testes usam `app.dependency_overrides` para mockar `get_db` e `get_current_client`
- HTTPBearer retorna 401 (não 403) quando Authorization header está ausente
- Supabase service_key bypassa RLS — correto para o backend
