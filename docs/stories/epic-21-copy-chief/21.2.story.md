## Dev Notes

### Contexto do sistema existente

`generate_copy_with_ai()` (copywriter.py:411+) é o ponto de entrada que monta o prompt final concatenando `_SYSTEM_PROMPT` + `_STATIC_LIBRARY` (que já inclui `_VIRAL_TRIGGERS`, `STRATEGY_PROMPTS`, `CONTENT_TYPE_PROMPTS`, `_VOICE_TONE_MAP`, `_RETRY_APPROACHES`) + o `user_message` dinâmico por request. A chamada real ao modelo acontece via Claude Sonnet (`MODEL = "claude-sonnet-4-6"`, copywriter.py:25) ou, se `COPY_PROVIDER=gemini`, via `_call_gemini_for_copy()` (copywriter.py:393) usando `gemini-2.5-flash`.

[Source: backend/app/agents/copywriter.py — lido diretamente]

### Restrições não-óbvias (não estão na epic, só no código de teste)

Estas restrições vêm diretamente dos testes existentes e **não podem ser quebradas silenciosamente**:

- `test_system_prompt_contains_algorithmic_directives` (test_copywriter_agent.py:279) exige que `_SYSTEM_PROMPT.lower()` contenha as substrings `hook`, `salvar`, `desconhecido`, `retenção`. Se a v2 remover ou reformular essas ideias a ponto de essas palavras sumirem, o teste quebra.
- `test_strategy_prompts_count` (test_copywriter_strategy.py:55) exige exatamente 23 entradas em `STRATEGY_PROMPTS`, distribuídas em 5 `feed_photo__*` + 6 `carousel__*` + 6 `reels__*` + 6 `story__*`. A story 21.1 pode reescrever o **texto** de qualquer entrada, mas não pode adicionar/remover chaves sem atualizar esses testes — e atualizar esses testes está fora do escopo combinado com o usuário (Modo A = só prompt engineering).
- `test_all_strategy_prompts_non_empty` — nenhuma entrada pode virar string vazia.

[Source: backend/tests/test_copywriter_agent.py:279-285]
[Source: backend/tests/test_copywriter_strategy.py:55-87]

### Provider duplo

Existem dois caminhos de geração: Claude Sonnet (padrão) e Gemini 2.5 Flash (`COPY_PROVIDER=gemini`, copywriter.py:393-408). Ambos consomem o mesmo `_SYSTEM_PROMPT` e `_STATIC_LIBRARY` — a v2 dos prompts vale para os dois providers automaticamente, não precisa de tratamento separado. O A/B desta story pode ser feito só com o provider padrão (Claude), a menos que a Story 21.1 tenha identificado diferença de comportamento entre providers.

[Source: backend/app/agents/copywriter.py:393-439]

### Risco a testar explicitamente no A/B

O Plano Mestre (`docs/plano-mestre-2026-07.md`, seção 3.2) documenta que a tendência 2026 é o Instagram despriorizar conteúdo "over-produced"/IA-genérico, com preferência por autenticidade. Gatilhos Sugarman mal aplicados podem soar "vendedor demais" para esse público. **O AC 6 desta story existe justamente para pegar esse risco antes de ir para produção** — não é opcional, é o critério que decide se a v2 é aprovada.

[Source: docs/plano-mestre-2026-07.md — seção "Risco Mitigation" do Epic 21]
