# Story 11.2 — CI Frontend (GitHub Actions)

**Epic:** 11 — Governança e Qualidade  
**Status:** Done  
**Criado em:** 2026-05-06  

## Descrição

Como desenvolvedor, quero que cada PR no repositório `autopost-frontend` verifique automaticamente TypeScript e build antes de permitir merge.

## Por quê

O frontend hoje não tem `lint` nem `typecheck` nos scripts do `package.json`. Podemos mergear TypeScript quebrado sem saber. O React Compiler experimental (`next.config.ts`) aumenta o risco.

## Acceptance Criteria

- [ ] Scripts `typecheck` e `build` adicionados ao `package.json`
- [ ] Workflow `.github/workflows/frontend-ci.yml` existe no repo
- [ ] Roda em todo `push` e `pull_request` para `main`
- [ ] Executa `tsc --noEmit` (typecheck)
- [ ] Executa `next build` (garante que a build não quebra)
- [ ] Falha bloqueia merge (via branch protection — Story 11.3)

## Implementação

**Arquivo:** `frontend/.github/workflows/frontend-ci.yml`  
**package.json scripts adicionados:** `typecheck`, `build` (já existe mas não estava no CI)
