# Story 11.4 — Hotfix Workflow AIOS

**Epic:** 11 — Governança e Qualidade  
**Status:** Done  
**Criado em:** 2026-05-06  

## Descrição

Como desenvolvedor, quero um workflow formal de hotfix no AIOS que seja rápido (15–20 min) mas rastreável, para uso em bugs críticos de produção sem pular o processo.

## Por quê

Bugs urgentes em produção criam pressão para pular tudo e "só corrigir". Sem um processo leve para urgências, o resultado é o que aconteceu hoje: correções sem story, sem QA, sem rastreabilidade. O hotfix workflow é o meio-termo entre "processo completo" e "Pedro resolve aí".

## Acceptance Criteria

- [ ] Arquivo `.aios-core/development/workflows/hotfix.yaml` criado
- [ ] Arquivo `.aios-core/development/tasks/hotfix-spec.md` criado
- [ ] Workflow define: documento do bug → spec da correção → implementação → smoke test → PR
- [ ] Task define template de 5 campos obrigatórios: sintoma, causa raiz, o que muda, como testar, rollback
- [ ] Referenciado em `.claude/rules/` para uso automático quando bug crítico é reportado

## Fluxo

```
Bug em produção reportado
  → @dev abre hotfix-spec (5 min)
  → @dev implementa
  → smoke test manual do caminho quebrado
  → PR aberto → CI roda → merge
  → @devops monitora deploy
```
