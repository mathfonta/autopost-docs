# Story 11.3 — Branch Protection

**Epic:** 11 — Governança e Qualidade  
**Status:** Done  
**Criado em:** 2026-05-06  

## Descrição

Como tech lead, quero que o branch `main` de ambos os repositórios exija PR aprovado com CI verde, eliminando o caminho de push direto.

## Por quê

Hoje qualquer agente (ou humano) pode fazer `git push origin main` diretamente, contornando qualquer processo. Branch protection é o enforcement no nível da plataforma — não depende de disciplina, é bloqueio técnico.

## Acceptance Criteria

- [x] `main` do `autopost-backend` exige PR para merge
- [x] `main` do `autopost-frontend` exige PR para merge
- [x] PR só pode ser mergeado com CI verde (checks das Stories 11.1 e 11.2)
- [x] Force push para `main` é bloqueado
- [x] Configuração feita via `gh api` (documentada e reproduzível)

## Nota sobre processo

Esta story deve ser implementada DEPOIS das Stories 11.1 e 11.2, pois o branch protection referencia os checks do CI pelo nome. Se ativada antes, bloquearia sem ter o CI para passar.
