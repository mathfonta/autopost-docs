# Story 11.1 — CI Backend (GitHub Actions)

**Epic:** 11 — Governança e Qualidade  
**Status:** Done  
**Criado em:** 2026-05-06  

## Descrição

Como desenvolvedor, quero que cada PR no repositório `autopost-backend` dispare automaticamente uma suite de verificações que impeçam o merge se algo estiver quebrado.

## Por quê

O NameError de `_VOICE_TONE_MAP` chegou à produção porque não existe nada que rode `import app.agents.copywriter` antes do deploy. Um CI de 2 minutos teria evitado 3 minutos de pipeline travado em produção.

## Acceptance Criteria

- [ ] Workflow `.github/workflows/backend-ci.yml` existe no repo
- [ ] Roda em todo `push` e `pull_request` para `main`
- [ ] Verifica import de todos os agentes (`analyst`, `copywriter`, `designer`, `publisher`)
- [ ] Executa `pytest tests/` completo
- [ ] Falha bloqueia merge (via branch protection — Story 11.3)

## Implementação

**Arquivo:** `backend/.github/workflows/backend-ci.yml`

**Jobs:**
1. `import-check` — `python -c "import app.agents.X"` para cada agente
2. `test` — `pytest tests/ -v --tb=short`

**Env vars no CI:** valores de teste (sem secrets reais — testes usam mocks)
