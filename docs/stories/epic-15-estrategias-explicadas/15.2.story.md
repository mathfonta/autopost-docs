# Story 15.2 — Cards de Estratégia Expandidos no SubStrategySelector

**Epic:** 15 — Estratégias Explicadas
**Status:** Done
**Agente executor:** @dev
**Data:** 2026-06-10
**Prioridade:** Alta
**Depende de:** Story 15.1 (metadados ricos disponíveis em strategies.ts)

---

## Contexto

Com os metadados ricos adicionados em 15.1, esta story os exibe no `SubStrategySelector`. O card atual mostra: ícone + label + desc + badge de objetivo. O usuário precisa de mais contexto antes de escolher — especialmente os campos `when_to_use`, `input_needed` e `output_example`.

A abordagem escolhida é o **card expansível**: por padrão o card é compacto (comportamento atual), e ao tocar um botão de expansão revela os novos detalhes. Sem redesenho total da tela — apenas adição de conteúdo expandido.

---

## Descrição

Modificar `SubStrategySelector.tsx` para que cada card de estratégia tenha um botão de expansão. Ao expandir:

1. **Quando usar** — linha curta com o contexto ideal
2. **O que você precisa** — inputs esperados (tipo de foto/vídeo)
3. **Exemplo de resultado** — frase de copy gerada por esta estratégia, em itálico

**Comportamento:**
- Apenas 1 card pode estar expandido por vez (accordion simples)
- Clicar no botão de seleção (área principal do card) seleciona a estratégia normalmente, sem precisar expandir
- O botão de expansão fica no canto superior direito do card (ícone `ChevronDown` / `ChevronUp`)
- Cards colapsados ficam idênticos ao design atual (sem regressão visual)
- Estado de expansão local ao componente (não persiste entre navegações)

**Referência visual (texto):**

```
┌────────────────────────────────────────────┐
│ [ícone]  Prova Social           [Confiança]│ ← card colapsado (atual)
│          Mostrar resultado ou entrega   [v]│
└────────────────────────────────────────────┘

┌────────────────────────────────────────────┐
│ [ícone]  Prova Social           [Confiança]│
│          Mostrar resultado ou entrega   [^]│ ← expandido
│ ──────────────────────────────────────── │
│ 📍 Quando usar                            │
│    Após entregar um serviço ou projeto     │
│                                            │
│ 📷 O que você precisa                     │
│    Foto do resultado final + cidade        │
│                                            │
│ ✏️ Exemplo de resultado                   │
│    "Mais um banheiro entregue em..."       │
│                                            │
│        [→ Usar esta estratégia]            │
└────────────────────────────────────────────┘
```

## Acceptance Criteria

- [ ] AC1: Cada card tem botão `ChevronDown`/`ChevronUp` no canto direito que expande/colapsa o detalhe
- [ ] AC2: Apenas 1 card expandido simultaneamente — expandir outro fecha o anterior
- [ ] AC3: Conteúdo expandido exibe `when_to_use`, `input_needed`, `output_example` com rótulos visuais distintos
- [ ] AC4: Clicar na área principal do card (fora do chevron) seleciona a estratégia e navega — comportamento idêntico ao atual
- [ ] AC5: Cards colapsados são visualmente idênticos ao design atual (sem regressão)
- [ ] AC6: `output_example` exibido em itálico e entre aspas, distinguindo-o como exemplo e não instrução
- [ ] AC7: `npm run build` sem erros TypeScript; nenhum `any` implícito introduzido

## Scope

### IN
- Modificação de `SubStrategySelector.tsx` — estado de expansão + layout expandido
- Leitura dos campos `when_to_use`, `input_needed`, `output_example` de `strategies.ts`

### OUT
- Nenhuma mudança no backend
- Nenhuma mudança em outros componentes (PostCard, ContentTypeBar, etc.)
- Não persistir o estado de expansão (localStorage/cookies) — sessional é suficiente
- Não adicionar animação complexa — `transition` CSS simples é suficiente

## Riscos

- Accordion pode conflitar com a área de toque do card em mobile → mitigação: separar claramente a área de toque do chevron vs. área de seleção da estratégia
- Cards muito longos quando expandidos podem ficar fora da tela em mobile → mitigação: scroll natural da página, não é um modal
