# Story 15.1 — Metadados Ricos das Estratégias (Frontend Config)

**Epic:** 15 — Estratégias Explicadas
**Status:** Draft
**Agente executor:** @dev
**Data:** 2026-06-10
**Prioridade:** Alta — bloqueante para 15.2

---

## Contexto

O arquivo `frontend/lib/strategies.ts` define as 23 estratégias de conteúdo. Cada uma tem: `id`, `label`, `desc` (12 palavras), `objective`, `icon`. Esse é o único dado que aparece para o usuário na hora de escolher o que postar.

O problema: `desc` é demasiado curto para o usuário entender o que está escolhendo. O documento de visão identificou que o usuário não entende o que cada estratégia faz, qual input precisa, qual resultado gera.

A solução desta story é enriquecer os metadados em `strategies.ts` sem tocar em nenhum componente ainda — isso fica para 15.2.

---

## Descrição

Adicionar 3 campos novos à interface `Strategy` em `strategies.ts` e preencher todos os 23 objetos:

```typescript
interface Strategy {
  id:           string;
  label:        string;
  desc:         string;          // mantido — usado nos cards atuais
  objective:    StrategyObjective;
  icon:         string;
  // NOVOS:
  when_to_use:     string;       // quando usar esta estratégia (1 linha, máx 80 chars)
  input_needed:    string;       // o que o usuário precisa ter na foto/vídeo (máx 80 chars)
  output_example:  string;       // exemplo real de linha de copy gerada (máx 120 chars)
}
```

**Conteúdo dos 23 registros (referência para implementação):**

### Feed Photo (5)

| id | when_to_use | input_needed | output_example |
|----|-------------|--------------|----------------|
| prova_social | Após entregar um serviço ou projeto concluído | Foto do resultado final + nome da cidade | "Mais um banheiro entregue em Joinville. Cada detalhe, nosso padrão." |
| ancora_de_marca | Quando quer reforçar o posicionamento da empresa | Foto com identidade visual da marca | "Qualidade não é sorte. É o nosso processo." |
| curiosidade_pergunta | Quando quer gerar comentários e debate | Qualquer foto relacionada ao nicho | "Você sabia que 70% das reformas atrasam por falta de planejamento?" |
| bastidores | Quando quer humanizar a marca e mostrar o processo | Foto da equipe ou do trabalho em andamento | "Por trás de cada entrega existe uma equipe que chegou cedo." |
| hero_shot | Quando tem uma foto de alta qualidade do produto/serviço | Melhor foto do trabalho, boa iluminação | "Acabamento premium. Do rascunho à entrega em 21 dias." |

### Carousel (6)

| id | when_to_use | input_needed | output_example |
|----|-------------|--------------|----------------|

| comparativo | Quando quer ajudar o cliente a decidir entre opções | Fotos de duas versões ou materiais | "Cerâmica ou porcelanato? Desliza e decide o que faz mais sentido." |
| checklist | Quando quer ser referência e gerar salvamentos | Foto de contexto do nicho | "Checklist da reforma: o que conferir antes de assinar o contrato." |

### Reels (6)

## Riscos

- Textos de `output_example` muito genéricos podem não representar o nicho real do cliente → mitigação: usar construção civil como referência (nicho da Espectra), os demais nichos vão variar pela geração do agente de qualquer forma
- Campo `output_example` pode não envelhecer bem se o copywriter mudar o estilo → mitigação: o campo é ilustrativo para o usuário, não é o output real
