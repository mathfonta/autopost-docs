# Story 5.2 — Retry: Nova Versão sem Reiniciar

**Status:** Ready for Review  
**Epic:** 5 — Fluxo de Aprovação Avançado  
**Story:** 5.2  
**Estimativa:** 3 pontos  
**Depende de:** Story 5.1 (Edição Inline — endpoint PATCH já existente)

---

## Descrição

Como cliente AutoPost, quero poder pedir uma nova versão da legenda diretamente na tela de aprovação — sem precisar rejeitar o post e reiniciar o fluxo — para ter mais opções antes de decidir publicar.

---

## Critérios de Aceitação

- [ ] AC1: Botão "Tentar novamente" (🔄) exibido na tela de aprovação ao lado de "Aprovar" e "Rejeitar"
- [ ] AC2: Ao clicar, exibe modal de confirmação: "Gerar nova versão? A legenda atual será substituída." com botões "Confirmar" e "Cancelar"
- [ ] AC3: Após confirmar, botão entra em estado loading ("Gerando...") e os botões Aprovar/Rejeitar ficam desabilitados
- [ ] AC4: Backend rechama o Agente Copywriter com a mesma foto e contexto — nova legenda gerada e salva no post
- [ ] AC5: Quando nova legenda estiver pronta, o card atualiza automaticamente via polling existente (5s)
- [ ] AC6: Contador de retries exibido discretamente: "Versão 2 de 3" — máximo 3 tentativas por post
- [ ] AC7: Após 3 tentativas, botão "Tentar novamente" some e mensagem "Máximo de versões atingido" aparece

---

## Escopo

**IN:**
- Botão retry na tela de aprovação
- Endpoint `POST /content-requests/{id}/retry` no backend
- Rechamada do Agente Copywriter com contexto original
- Contador de tentativas (máx 3)
- Feedback visual durante processamento

**OUT:**
- Retry do Agente Designer (imagem) — só a legenda é regerada
- Retry na tela de Histórico
- Escolha de "estilo" diferente no retry (isso é o Menu de Intenção — Story 5.3)

## Dev Notes

**Endpoint sugerido:**
```
POST /content-requests/{id}/retry
Response 200: { "id": ..., "retry_count": 2, "status": "generating_copy" }
Response 422: { "detail": "Max retries reached" } se retry_count >= 3
Response 409: { "detail": "Not in awaiting_approval status" }
```

**Fluxo Celery:**
```
retry endpoint → não rechama analyze_photo
               → rechama generate_copy(analysis_result=existing)
               → generate_copy atualiza caption → status volta a awaiting_approval
```
O polling do frontend (5s) detecta a mudança de status automaticamente.
