# Story 6.3 — Frontend Multi-Foto

**Status:** Ready  
**Epic:** 6 — Conteúdo Rico: Múltiplas Fotos, Antes/Depois e Carrossel  
**Story:** 6.3  
**Estimativa:** 3 pontos  
**Depende de:** Story 6.1 (endpoint aceita múltiplas fotos)  
**Pode ser paralela a:** Story 6.2 (pipeline é independente do frontend)

---

## Descrição

Como cliente AutoPost, quero selecionar múltiplas fotos na tela principal e ver uma prévia de todas antes de enviar — para que eu possa verificar a seleção, remover fotos indesejadas e escolher o tipo de conteúdo adequado (antes/depois ou carrossel) sem precisar reiniciar o fluxo.

---

## Critérios de Aceitação

- [ ] **AC1 — FAB aceita múltipla seleção:** O botão de câmera (FAB) suporta seleção de 1–10 fotos. No mobile, o bottom sheet exibe "Escolher da galeria (múltiplas)" como opção. No desktop, o file picker abre com `multiple` habilitado.
- [ ] **AC2 — Preview antes do envio:** Após selecionar 2+ fotos, o usuário vê uma fila de thumbnails (imagens 60×60px) em linha horizontal com um "×" para remover cada foto individualmente — antes de confirmar o upload.
- [ ] **AC3 — Upload multi-foto via FormData:** `POST /content-requests` é chamado com `FormData` contendo N entradas `photos` (uma por arquivo). O campo legado `photo` não é usado na nova implementação.
- [ ] **AC4 — Menu de Intenção exibe novos tipos:** Quando 2+ fotos são selecionadas, o `IntentMenu` exibe as opções `Antes/Depois` e `Carrossel` além das existentes. Com 1 foto, apenas os tipos simples são exibidos (comportamento atual).
- [ ] **AC5 — `content_type` enviado no FormData:** O valor selecionado no `IntentMenu` é incluído como campo `content_type` no FormData.
- [ ] **AC6 — `PostCard` exibe thumbnails múltiplos:** Quando `photo_urls` tem 2+ itens, o card exibe uma faixa de thumbnails (máx 3 visíveis + indicador "+N") ao invés da foto única. O clique abre o carrossel de preview em tela cheia (modal simples com setas).
- [ ] **AC7 — Retrocompat visual:** Cards com foto única continuam exibindo a foto como antes (`photo_url`). Nenhuma regressão visual nos cards existentes.
- [ ] **AC8 — Validação no cliente:** Antes de enviar, validar: máx 10 fotos (erro inline), `before_after` = exatamente 2 (aviso antes de enviar), `carousel` = 2–10 (aviso antes de enviar).
- [ ] **AC9 — Build limpo:** `npm run build` e `npm run typecheck` passam sem erros. Todos os tipos TypeScript corretos.
- [ ] **AC10 — Spinner e error handling:** Durante upload multi-foto, spinner é exibido por foto (ex: "Enviando 2/3..."). Em caso de erro, mensagem clara sem dados residuais no estado.

### FormData Multi-foto

```typescript
async function submitPhotos(files: File[], contentType?: string) {
  const formData = new FormData();
  files.forEach(file => formData.append("photos", file));
  if (contentType) formData.append("content_type", contentType);
  
  const res = await apiClient.post("/content-requests", formData);
  return res.data;
}
```

**Atenção:** Não definir `Content-Type` header manualmente — o browser define automaticamente com boundary para multipart.

### PhotoPreviewQueue — componente simples

```tsx
export function PhotoPreviewQueue({ files, onRemove }: {
  files: File[];
  onRemove: (index: number) => void;
}) {
  return (
    <div className="flex gap-2 flex-wrap my-2">
      {files.map((file, i) => (
        <div key={i} className="relative">
          <img
            src={URL.createObjectURL(file)}
            className="w-16 h-16 object-cover rounded"
          />
          <button
            onClick={() => onRemove(i)}
            className="absolute -top-1 -right-1 bg-red-500 text-white rounded-full w-4 h-4 text-xs flex items-center justify-center"
          >×</button>
        </div>
      ))}
      <span className="text-sm text-gray-500 self-center">
        {files.length} foto(s)
      </span>
    </div>
  );
}
```

**Atenção:** Revogar URLs após uso: `useEffect(() => () => files.forEach(f => URL.revokeObjectURL(f.name)), [files])` — na prática usar `URL.createObjectURL(file)` inline com cleanup adequado.
