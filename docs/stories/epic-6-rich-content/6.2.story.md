## Descrição

Como cliente AutoPost, quero que o pipeline processe automaticamente múltiplas fotos e gere uma legenda contextualizada ao tipo de conteúdo (antes/depois ou carrossel) — para que eu não precise adaptar manualmente o texto e o Instagram receba um carrossel nativo, não apenas uma foto.

Como desenvolvedor, preciso adaptar as 4 tasks Celery do pipeline (`analyze_photo`, `generate_copy`, `prepare_design`, `publish_post`) para operar em modo multi-foto quando `photo_keys` contém mais de um elemento, mantendo o fluxo single-photo 100% inalterado.

---

## Critérios de Aceitação

- [ ] **AC1 — `analyze_photo` itera sobre `photo_keys`:** Quando `len(photo_keys) > 1`, o agente Analista executa análise individual para cada foto e armazena os resultados em `analysis_result["photos"]` (lista). Para foto única, o resultado permanece em `analysis_result` sem quebras.
- [ ] **AC2 — `generate_copy` recebe análises múltiplas e `content_type`:** O Copywriter recebe o array de análises + `content_type` no prompt. Para `before_after`: gera legenda narrativa comparativa (antes → depois). Para `carousel`: gera legenda principal + descrição de sequência de slides. Para foto única: comportamento atual preservado.
- [ ] **AC3 — `prepare_design` processa cada foto:** O Designer chama Pillow uma vez por foto em `photo_keys`, gera uma imagem 1080×1080 para cada, e armazena as chaves de design em `design_result["design_keys"]` (lista). Para foto única: `design_result` permanece como está.
- [ ] **AC4 — `publish_post` detecta carrossel:** Se `len(photo_keys) > 1`, usa o fluxo carrossel da Meta Graph API:
    1. Cria container de mídia individual para cada imagem (`POST /{ig-user-id}/media`)
    2. Polling `status_code=FINISHED` em cada container (timeout: `N * 60s`)
    3. Cria carousel container (`media_type=CAROUSEL`)
    4. Publica carousel container
- [ ] **AC5 — Fluxo single-photo inalterado:** Para `content_type` sem `before_after` ou `carousel`, ou quando `photo_keys` tem 1 elemento, o pipeline executa exatamente como antes. Nenhum dado existente é afetado.
- [ ] **AC6 — Regressão zero em testes existentes:** Todos os 117 testes existentes continuam passando após as mudanças no pipeline.
- [ ] **AC7 — Testes de pipeline multi-foto:** Mínimo 15 novos testes cobrindo: análise multi-foto, copy `before_after`, copy `carousel`, design multi-foto, publicação carrossel (com mock da Meta API), polling de containers, timeout proporcional, rollback em falha de container.
- [ ] **AC8 — Logging adequado:** Cada etapa do pipeline multi-foto loga `request_id`, `n_photos`, e resultado de cada iteração. Erros em fotos individuais são logados sem interromper o pipeline (best-effort para design).

## Dev Notes

### Detecção de Modo Multi-foto no Pipeline

Padrão a usar em todas as tasks:

```python
photo_keys = req.photo_keys or [req.photo_key]
is_multi = len(photo_keys) > 1
```

Isso garante retrocompat: se `photo_keys` for `None` (registro antigo), usa `photo_key` singular.

### Prompt do Copywriter — Before/After

```python
if content_type == "before_after":
    prompt = f"""
    Você está gerando legenda para um post ANTES/DEPOIS no Instagram.
    
    FOTO 1 (ANTES): {analyses[0]}
    FOTO 2 (DEPOIS): {analyses[1]}
    
    Gere uma legenda narrativa que:
    - Começa descrevendo o estado "antes" brevemente
    - Enfatiza a transformação/progresso alcançado
    - Inclui CTA relacionado ao serviço da empresa
    - Tom: profissional, inspirador, específico ao setor
    - Máximo 300 palavras
    """
```

### Prompt do Copywriter — Carousel

```python
if content_type == "carousel":
    analyses_text = "\n".join([f"Slide {i+1}: {a}" for i, a in enumerate(analyses)])
    prompt = f"""
    Você está gerando legenda para um CARROSSEL no Instagram com {n} fotos.
    
    ANÁLISE DAS FOTOS (em sequência):
    {analyses_text}
    
    Gere:
    1. Legenda principal do post (para o slide 1) — max 150 palavras
    2. Descrição da sequência — o que o usuário vê ao avançar os slides
    
    Retorne em JSON: {{"caption": "...", "sequence_description": "..."}}
    """
```

### Meta API Carousel — Referência

```
# Step 1: criar containers de item
POST /{ig-user-id}/media
  ?image_url=https://...
  &is_carousel_item=true
  &access_token={token}
→ { "id": "container_id_1" }

# Step 2: polling por container
GET /{container_id}?fields=status_code&access_token={token}
→ { "status_code": "FINISHED" }  # aguardar antes de continuar

# Step 3: criar carousel container
POST /{ig-user-id}/media
  ?media_type=CAROUSEL
  &children={id1},{id2}
  &caption={legenda_url_encoded}
  &access_token={token}
→ { "id": "carousel_container_id" }

# Step 4: publicar
POST /{ig-user-id}/media_publish
  ?creation_id={carousel_container_id}
  &access_token={token}
→ { "id": "instagram_post_id" }
```

## Riscos

| Risco | Probabilidade | Mitigação |
|-------|--------------|-----------|
| Regressão no pipeline single-photo | Média | Testes de regressão obrigatórios; usar `photo_keys or [photo_key]` pattern |
| Timeout no polling de N containers | Média | Timeout proporcional `N * 60s`; log explícito por container |
| Meta API rate limit em uploads sequenciais | Baixa | `asyncio.sleep(1)` entre criações de container |
| Claude Haiku sem contexto entre análises de fotos | Baixa | Análise independente por foto; Copywriter recebe todas as análises |
