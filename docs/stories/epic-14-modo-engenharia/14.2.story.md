# Story 14.2 — Sequência de Ataque Editorial para Clientes Novos

## Contexto

Um cliente novo no AutoPost começa com zero histórico. O algoritmo do Instagram precisa de um período de "teste" para entender quem é o público do perfil. Postar de forma aleatória desperdiça esse período. O Modo Engenharia define uma **sequência de ataque** de 10 posts com objetivo algorítmico crescente: do simples (retenção) ao ousado (viralização).

Hoje o AutoPost não tem lógica de sequência — cada post é gerado independentemente. Esta story adiciona um campo `is_new_client` que ativa a sequência de ataque, e um card "Plano de Ataque" no dashboard para os primeiros 10 posts.

## Descrição

Quando um cliente gera o **primeiro post** (ou a conta tem menos de 10 posts publicados), o sistema:

1. Detecta que é cliente novo via contagem de posts publicados no banco
2. Sugere o **próximo passo da sequência** no dashboard como card contextual
3. Injeta no copywriter o **objetivo algorítmico do post atual** (ex: "Este post deve ser otimizado para salvamentos — use formato de lista")

### Sequência de ataque (baseada no Modo Engenharia)

| Post # | Formato sugerido | Objetivo algorítmico | Injeção no copywriter |
|--------|-----------------|---------------------|----------------------|
| 1 | Reels educativo simples | Retenção — teste inicial | "Foque em retenção: hook direto, sem jargão, entrega clara" |
| 2 | Carrossel lista (5-7 itens) | Salvamentos | "Formato lista gera save automaticamente — use numeração" |
| 3 | Reels com gancho opinativo | Comentários e shares | "Opinião forte gera comentário — tome um lado claro" |
| 4 | Post estático aspiracional | Qualificação de audiência | "Imagem + texto curto aspiracional — filtra público certo" |
| 5 | Reels antes/depois | Replays + permanência | "Visual de transformação incentiva replay — resultado claro" |
| 6-10 | Mix definido pelo sistema | Consolidação | Baseado nos formatos que tiveram melhor performance nos posts 1-5 |

## Acceptance Criteria

- [x] AC1: Campo `attack_sequence_position` (int, default 0) adicionado à tabela `clients` via migration Alembic
- [x] AC2: `attack_sequence_position` é incrementado automaticamente a cada post publicado com sucesso (status → `published`), até o máximo de 10
- [x] AC3: Quando `attack_sequence_position < 10`, o copywriter recebe injeção `#OBJETIVO_SEQUENCIA` com a diretiva do post atual (tabela acima)
- [x] AC4: Card "Plano de Ataque" exibido no dashboard quando `attack_sequence_position < 10`: mostra posição atual, objetivo do próximo post e barra de progresso (X/10)
- [x] AC5: Card desaparece automaticamente quando `attack_sequence_position >= 10`
- [x] AC6: Testes unitários cobrem: incremento do position, injeção correta por position, ausência de injeção após position 10

## Scope

### IN
- Migration Alembic: campo `attack_sequence_position` em `clients`
- Lógica de incremento no pipeline de publicação (`publish_post` task)
- Injeção do objetivo no copywriter quando position < 10
- Card "Plano de Ataque" no frontend dashboard
- Testes unitários para toda a lógica

### OUT
- Não bloquear geração de post se sequência não for seguida (usuário pode postar na ordem que quiser)
- Não alterar o conteúdo aprovado pelo usuário — a injeção é só para o copywriter na geração
- Nenhum onboarding forçado ou tutorial novo
- Posts 6-10 com mix inteligente ficam para v2 — usar formato livre neste sprint

- Card no dashboard pode poluir a interface → mitigação: card apenas visível quando há posts pendentes de aprovação ou nas primeiras 2 semanas de conta
