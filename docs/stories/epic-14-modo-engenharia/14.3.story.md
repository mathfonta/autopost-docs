## Contexto

O card de streak (`StreakBar`) existe no dashboard desde a Epic 4, mas estava alimentado por dados mockados hardcoded (`STREAK_MOCK`): todo usuário via "3 dias seguidos" independente da realidade. Esta story substitui o mock por um endpoint real que calcula dias consecutivos e progresso semanal a partir de posts efetivamente publicados.

## User Story

**As a** usuário do AutoPost,
**I want** ver meu streak real de dias seguidos publicando,
**so that** tenha um incentivo concreto e honesto para manter consistência.

## Acceptance Criteria

- [x] AC1: `GET /insights/streak` retorna `{streak, week_days, week_goal}` com dados reais do client autenticado
- [x] AC2: `week_days` é array de 7 booleans (Seg→Dom) indicando se houve post publicado em cada dia da semana atual
- [x] AC3: `streak` conta dias consecutivos com publicação retroativamente a partir de hoje (se publicou hoje, conta; ontem, conta, etc.)
- [x] AC4: `week_goal` fixo em 5 posts/semana
- [x] AC5: Frontend substitui `STREAK_MOCK` por chamada real ao endpoint; card some se endpoint falhar (sem crash)
- [x] AC6: Testes cobrem: semana com 3 dias, streak zero, streak de semanas consecutivas

## Scope

### IN
- `GET /insights/streak` — endpoint autenticado, tenant-safe
- `StreakResponse` schema
- Substituir mock no frontend por fetch real
- Fallback: se endpoint falhar, esconder o card (não mostrar dado errado)

### OUT
- Configuração de meta semanal pelo usuário (fica hardcoded em 5)
- Notificações push de streak (v2)
- Streak baseado em aprovações ou criações (só publicações contam)

## Estimativa

**1 ponto** (endpoint simples + substituição do mock)

## Riscos

| Risco | Probabilidade | Mitigação |
|-------|-------------|-----------|
| Timezone do servidor (UTC) vs timezone do cliente → streak pode aparecer 1 dia adiantado/atrasado | Média | Usar `date` baseado em `published_at` truncado para dia no banco — consistente para todos os clientes independente de TZ local |
| Cliente sem nenhum post publicado → streak zero deve ser tratado graciosamente | Alta (clientes novos) | Implementado: `streak=0`, `week_days=[False]*7`, `week_goal=5` — card aceita streak zero sem crash |
| Falha no endpoint esconde o card sem notificar o usuário | Baixa | Comportamento intencional (definido no AC5 como fallback aceitável para MVP) |
